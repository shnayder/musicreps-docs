# Update inventory docs when changing a system

When a PR or session changes a system that has an inventory doc in `inventory/`,
update the inventory as part of the same session. The inventory should reflect
the current state of the code after the change.

If no inventory doc exists yet for the system being changed, note the gap but
don't create one — that's a separate task.
