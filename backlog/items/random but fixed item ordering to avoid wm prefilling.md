# Random-but-fixed item ordering to avoid WM prefilling

For at least some skills, items should have a fixed but randomized order
(stable per-user, randomized once) instead of a deterministic order. The
goal is to mix up sequences so the user doesn't prefill working memory from
the previous similar question.
